## Source Boundaries

The OCP GitOps page is an overview. It does not provide detailed Operator
installation procedures, channels, namespaces, Argo CD instance fields,
Application or AppProject schema, SSO configuration, RBAC, cluster role
configuration, notifications, repository credentials, sync options, or resource
tracking fields.

Because Red Hat OpenShift GitOps releases on a different cadence from OCP,
detailed behavior must be captured from the separate Red Hat OpenShift GitOps
documentation set after a GitOps product version is selected for the project.
