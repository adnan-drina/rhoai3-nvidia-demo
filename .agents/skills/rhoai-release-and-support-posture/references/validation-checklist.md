# Validation Checklist

Use this checklist before finalizing release or support claims.

## Source Freshness

- Release notes URL matches the active baseline.
- Lifecycle page is rechecked for support dates and release type guidance.
- Known issues and support removals are reviewed for affected demo features.

## Claim Review

- GA, Technology Preview, Developer Preview, deprecated, removed, EUS, and
  Early Access terms are used only when backed by Red Hat sources.
- Preview features are not described as production supported.
- Developer Preview and Early Access features are not recommended for
  business-critical workloads.
- Any demo use of unsupported or preview features is visibly labeled.

## Handoff

- Operator channel decisions route to `rhoai-update-channels`.
- API compatibility claims route to `rhoai-api-tiers`.
- Component behavior routes to the relevant component skill.
